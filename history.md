### 2018-04-14 ###
puyd:
1. 系统授权首先区分功能模块。模块下可定义授权项目。

2. 授权项目分成四类：
2.1 登陆可见——会员登录后即可使用，比如会员资料菜单组下的各项功能
2.2 服务授权——用户需要授权之后才可以使用，比如专利检索等
2.3 管理功能——仅后台管理用户可以使用，比如文章发布等功能
2.4 内部功能——系统需要的内部功能设定，菜单上不可见

3. 授权项目的计分方式分成三类：
3.1 不计费
3.2 按期计费
3.3 按次计费

4. 用户角色分类
4.1 一般用户，按服务类功能授权使用，且按期或按次扣费
4.2 内部用户，按服务类功能授权使用，忽略项目的计费方式，不计费
4.3 内部管理，按管理类功能授权使用
4.4 系统管理，可以使用所有管理类功能

### 2018-04-18 ###
1. 权限: 包含一系列功能组,每个功能组必须要有名称和图例,并包含一系列功能列表.
    [
        {
            group:"系统维护",                                //组名称
            img:"url('assets/images/desk/cell06.jpg')",     //图例
            func:[                                          //功能列表
                {
                    id: ?,                                  //类型待定
                    label: "用户管理",                        //名称
                    href: "#/sysmain/accountmng",           //页面链接
                    kind: 0,                                //功能分类,'0-登录可见','1-服务授权','2-系统管理','3-内部功能'
                    pay: 0,                                 //计费方式,'0-不计费','1-按期计费','2-按次计费'
                },
                ....
            ]
        },
        ....
    ]
    

2. 角色: 一个角色必须有名称和类型,自定义的角色包含一组相关功能的列表,列表中的功能可用于给相关用户授权,不在列表中的功能则禁止授权与使用
    [
        {
            id:0,                  //id
            label:'一般用户',       //角色名称
            func:[]                //预定义角色的功能列表为空,按系统规则处理
        },
        {   id:1,
            label:'内部用户',
            func:[]
        },
        {
            id:2,
            label:'网站维护',
            func:[]
        },
        {
            id:3,
            label:'系统管理',
            func:[]
        },
        {
            id:101,                  //自定义角色
            label:'自定义角色',
            func:[ ?,?,?,... ]       //功能列表为相关功能的id数组
        },
        ......
    ]

3. 用户: 一个用户必须有名称,并至少对应一个角色,默认角色为一般用户,并包含一系列功能授权
    [
        {
            id: 101,                    //id
            label: "puyd"               //名称
            pwd: "123456"               //登录密码
            code: "13511650262"         //手机号
            mail: "admin@aiptan.com"    //邮件地址
            role: [ 0, 1, 2, 3 ]        //角色,id数组
            func: [                     //功能授权
                {
                    id: 1,              //授权功能的id
                    active: true,       //是否启用
                    count:  99999,      //剩余点数，仅针对记次的功能项目
                    stop: "2099-12-31"  //终止日期，仅针对按期计费的功能项目
                },
                ...
            ]
        },
        ......
    ]
